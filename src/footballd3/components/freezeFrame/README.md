# freezeFrame

Static 360 freeze-frame snapshot rendered as a D3 overlay on an existing `pitch.js` instance.

Displays player positions and the ball for a single instant of play. Players are encoded by role:

| Marker | Meaning |
|--------|---------|
| Filled circle (navy `#1E3A5F`) | Teammate (field player) |
| Filled circle (red `#9F1239`) | Opponent (field player) |
| Diamond polygon (team colour) | Goalkeeper — same fill as their team side |
| White outer ring | Actor — the player who performed the action |
| White circle with dark stroke | Ball |

An optional broadcast field-of-view polygon (`visible_area`) can be toggled via `config.showVisibleArea`. It is off by default.

The component exposes an `update(frameData)` method so callers can swap between snapshots (e.g., cycling through goals) without re-rendering the pitch.

## JSON contract

**File:** `sample_data/freeze_frames_{match_id}_goals.json`

```json
{
  "goals": [
    {
      "ball": { "x": 116.0, "y": 37.4 },
      "frame": [
        { "x": 108.1, "y": 34.5, "teammate": false, "actor": false, "keeper": true },
        { "x": 114.3, "y": 36.9, "teammate": true,  "actor": true,  "keeper": false }
      ],
      "visible_area": [88.2, 0.0, 120.0, 0.0, 120.0, 80.0, 88.2, 80.0],
      "metadata": {
        "match_id": 3943043,
        "event_id": "a1b2c3d4-...",
        "display_name": "Nico Williams",
        "team":     "Spain",
        "action_type": "Shot",
        "minute":   47,
        "competition": "UEFA Euro 2024",
        "match_label": "Spain vs England"
      }
    }
  ],
  "match_metadata": {
    "match_id": 3943043,
    "competition": "UEFA Euro 2024",
    "match_label": "Spain vs England"
  }
}
```

| Field | Notes |
|-------|-------|
| `ball.x`, `ball.y` | StatsBomb-native 120×80 coordinates — untransformed. Pitch handles pixel mapping via `px()`. |
| `frame[].x`, `.y` | Player position in StatsBomb coordinates. |
| `frame[].teammate` | `true` = same team as the actor; `false` = opposing team. |
| `frame[].actor` | `true` = the player who performed the action (e.g., the shooter). At most one per frame. |
| `frame[].keeper` | `true` = goalkeeper, rendered as a diamond instead of a circle. |
| `visible_area` | Flat polygon array `[x1, y1, x2, y2, ...]` in StatsBomb space. The FOV polygon covers only the broadcast camera's field of view; not all 22 players may be visible. |
| `metadata` | Not rendered by the D3 component. The `display_name`, `team`, and `minute` fields are consumed by the index.html goal-navigation label. `display_name` is the player nickname (or full name when no nickname exists), resolved Python-side. |

Generated by: `src/statsbomb/extract_freeze_frame.py`

## Usage

```javascript
import { createPitch }       from "../pitch/pitch.js";
import { createFreezeFrame } from "../freezeFrame/freezeFrame.js";

// 1. Fetch the goals data.
const data = await fetch("sample_data/freeze_frames_3943043_goals.json").then(r => r.json());

// 2. Create the pitch (caller owns this step).
const pitch = createPitch(d3.select("#my-svg"), {
  mode:        "full",
  orientation: "horizontal",
  pxPerYard:   7,
  theme:       "whiteboard",
});

// 3. Overlay the first goal's freeze frame.
const { update } = createFreezeFrame(pitch, data.goals[0], {
  showVisibleArea: false,      // toggle the FOV polygon
  mirrorX:         true,       // required when pitch uses mode:"half" — reflects x as 120-x
  teamColor:       "#1E3A5F", // teammate marker fill
  opponentColor:   "#9F1239", // opponent marker fill
  markerRadius:    6,          // base player circle radius (px)
});

// 4. Swap to the next goal (pitch stays; freeze-frame layer transitions).
update(data.goals[1]);
```
