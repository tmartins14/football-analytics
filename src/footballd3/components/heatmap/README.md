# heatmap

On-ball event density surface for a single player in a single match, rendered as a D3 overlay on an existing `pitch.js` instance.

## What this shows — and what it does not

**What it shows:** where the player was *involved in play*. Each event where the player was the actor and a spatial location is recorded (passes, shots, carries, pressures, duels, ball receipts, etc.) contributes to the surface. High-density zones are where the player touched the ball most often.

**What it does not show:** where the player *physically was* throughout the match. StatsBomb 360 freeze-frame data captures all visible players at the moment of specific events, but it does not carry individual player identity for off-ball tracking. Continuous per-player position data is not available in the StatsBomb open dataset. Do not label this component "player movement" or "player positioning" — it is **on-ball involvement only**.

## The density surface depends on a smoothing choice

The heatmap is **not a parameter-free fact**. It is a Gaussian KDE surface evaluated with an explicit bandwidth parameter (`bandwidth_yards`). A smaller bandwidth reveals finer local clusters; a larger one blurs nearby clusters together into broader zones. The bandwidth used to produce the JSON is always recorded in `metadata.bandwidth_yards` — inspecting the file tells you exactly how the surface was computed. Default: **5.0 yards** (StatsBomb native coordinate unit).

## JSON contract

**File:** `sample_data/heatmap_{match_id}_{player_slug}.json`

```json
{
  "grid": {
    "cols": 60,
    "rows": 40,
    "values": [
      [0.0, 0.03, 0.11, ...],
      ...
    ]
  },
  "metadata": {
    "match_id": 3943043,
    "display_name": "Rodri",
    "team": "Spain",
    "competition": "UEFA Euro 2024",
    "match_label": "Spain vs England",
    "event_count": 67,
    "method": "kde",
    "bandwidth_yards": 5.0,
    "grid_cols": 60,
    "grid_rows": 40,
    "pitch_width_yards": 120,
    "pitch_height_yards": 80
  }
}
```

| Field | Notes |
|---|---|
| `metadata.display_name` | Player nickname (or full name when no nickname exists), resolved Python-side. Displayed in the gallery label. |
| `grid.values` | 2D array `[rows][cols]` of floats normalised to `[0.0, 1.0]` relative to the grid maximum. The renderer maps this directly to colour and opacity. |
| `grid.cols`, `grid.rows` | Grid dimensions. Each cell covers `(120/cols) × (80/rows)` yards. Default 60×40 = 2-yard cells. |
| `metadata.bandwidth_yards` | KDE bandwidth in StatsBomb yards. **Consumed by documentation and tooling only — the renderer does not use it.** Always inspect this field before interpreting the surface. |
| `metadata.event_count` | Number of on-ball events used to fit the KDE. Displayed in the gallery label. |
| `metadata.method` | Always `"kde"` for this extractor. Present for traceability. |
| `metadata.pitch_width_yards`, `pitch_height_yards` | Canonical 120×80 — recorded for reference, not read by the renderer. |

Generated by: `src/statsbomb/extract_heatmap.py`

## Usage

```javascript
import { createPitch }   from "../pitch/pitch.js";
import { createHeatmap } from "../heatmap/heatmap.js";

// 1. Fetch the heatmap data.
const data = await fetch("sample_data/heatmap_3943043_rodri.json").then(r => r.json());

// 2. Create the pitch (caller owns this step).
const pitch = createPitch(d3.select("#my-svg"), {
  mode:        "full",
  orientation: "horizontal",
  pxPerYard:   7,
  theme:       "whiteboard",
});

// 3. Overlay the heatmap. It renders under pitch markings automatically.
const { update } = createHeatmap(pitch, data, {
  renderStyle: "smooth",    // "smooth" (canvas bilinear) or "raster" (SVG rect per cell)
  colorLow:    "#FAF7F0",   // zero-density color — matches whiteboard background
  colorHigh:   "#9F1239",   // peak-density color — focal accent
  maxOpacity:  0.85,        // pitch markings stay visible at full density
});

// 4. Swap to a different player's grid without re-rendering the pitch.
const data2 = await fetch("sample_data/heatmap_3943043_declan_rice.json").then(r => r.json());
update(data2);
```

## Render styles

| `renderStyle` | Mechanism | Visual result |
|---|---|---|
| `"smooth"` (default) | Off-screen canvas with bilinear interpolation → SVG `<image>` | Continuous gradient surface, no grid seams |
| `"raster"` | One SVG `<rect>` per grid cell (2400 at 60×40 default) | Visible cell grid; useful for inspecting raw KDE output |

## Re-running the extractor

```bash
uv run python src/statsbomb/extract_heatmap.py
```

The extractor resolves the Euro 2024 Final live (no hardcoded IDs), picks the player with the most on-ball events, and writes the JSON to `src/footballd3/sample_data/`. To use a specific player, call `extract_player_events(match_id, player_name="Full Name")` directly.
