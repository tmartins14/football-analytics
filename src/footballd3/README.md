# footballd3 — component gallery

Local development harness for the footballd3 component library. Open the gallery
in a browser, edit a component file, refresh.

## Running

From the repo root:

```bash
python3 -m http.server 8000
```

Then open: `http://localhost:8000/src/footballd3/index.html`

The server is required because ES modules don't load over `file://`.

## Structure

```
src/footballd3/
  index.html                      ← gallery harness
  pitch.js                        ← pitch component (root-level, used by index.html)
  shot_map.js                     ← shot map component (root-level, used by index.html)
  components/
    pitch/
      pitch.js                    ← pitch component source
      README.md                   ← pitch docs
    shotMap/
      shotMap.js                  ← shot map component source
      README.md                   ← shot map docs
  sample_data/
    shots_3943043.json            ← UEFA Euro 2024 Final shots (generated by statsbomb package)
    sample_corners.json           ← corners fixture
../style/
  style.css                       ← design tokens (CSS variables, Google Fonts)
```

## Components

| Component | File | Description |
|---|---|---|
| **pitch** | `components/pitch/pitch.js` | Renders a StatsBomb 120×80 pitch into a D3 SVG selection. Returns `{ g, px }` for downstream components. |
| **shotMap** | `components/shotMap/shotMap.js` | Overlays shots on a half-pitch. Circle area encodes xG; color encodes outcome. Hover tooltip shows player / outcome / minute / xG. |

Full API docs in each component's README.

## Adding a new component

1. Create `components/<name>/<name>.js`. Export a function that accepts a D3
   selection (or the object returned by `createPitch`) plus a config object.
2. Add a `components/<name>/README.md` covering what it does, the JSON contract,
   and a minimal usage example (required by the documentation standard).
3. Add a root-level alias (`<name>.js`) if the gallery harness needs to import it.
4. Wire it into `index.html`: add a `<section>` with a stage SVG, import the
   module, fetch the sample data, and call the render function.

## Notes

- `sample_data/` holds real StatsBomb data exported by the Python extractors in
  `src/statsbomb/`. Never fabricate fixture data.
- This harness is for component development, not publication. A publication
  stack (Quarto / Astro / Observable) is deferred until the first analysis is
  substantively complete.
